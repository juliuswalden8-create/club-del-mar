Reservation system

Reserve a table with a cleaner, more discreet booking flow.

Each booking is saved as structured data while the interface stays calm, minimal and easy to scan.

Bookings are saved to the backend and available from /api/bookings.

Backend handoff

Reservations move directly into a readable JSON feed.

The form posts into the backend, each reservation gets a reference and timestamp, and the feed can be protected with an admin key when you go live.

POST /api/bookings
GET /api/bookings
  • Every reservation is stored with guest details, party size, notes and a created timestamp.
  • Your internal tools can fetch bookings back as JSON from the same backend endpoint.
  • Set BOOKINGS_ADMIN_KEY before production to protect the booking feed.

For local demos the feed can stay open, but production should use an admin key and a real database.
